# 功能研究 - 响应式模板引擎

## Table Stakes（基础功能）

这些功能是任何响应式模板引擎必须具备的：

### 核心指令
- **x-if** - 条件渲染
  - 当表达式为 truthy 时渲染元素
  - 为 falsy 时移除元素
  - 支持嵌套使用

- **x-for** - 列表渲染
  - 遍历数组或对象
  - 支持 `item of items` 语法
  - 支持 `key` 属性标识唯一性

- **x-text** - 文本内容绑定
  - 将表达式的值作为文本内容
  - 自动转义 HTML 特殊字符

- **x-html** - HTML 内容绑定
  - 将表达式的值作为 HTML 内容
  - 不转义，需注意 XSS 风险

- **x-visible** - 显示/隐藏控制
  - 通过 `display` 属性控制元素可见性
  - 元素始终存在于 DOM 中

### 事件绑定
- **x-click** - 点击事件
- **x-input** - 输入事件
- **x-submit** - 表单提交
- **通用事件** - x-on:event 语法

### 属性绑定
- **x-bind:attr** - 属性值绑定
- **:attr 简写** - 便捷语法
- **布尔属性** - 支持 disabled、readonly 等

### 类和样式绑定
- **x-class** - 动态类名
  - 对象语法：`{active: isActive}`
  - 数组语法：`['class1', class2]`
  - 字符串语法

- **x-style** - 动态样式
  - 对象语法：`{color: textColor}`
  - 字符串语法

### 核心机制
- **表达式解析** - 支持简单的 JavaScript 表达式
- **依赖追踪** - 自动追踪指令中使用的状态
- **细粒度更新** - 只更新受影响的 DOM 节点
- **初始化扫描** - 启动时扫描模板中的指令

## Differentiators（差异化功能）

基于 AutoStore 的独特优势：

### 异步状态支持
- **加载状态反馈** - 异步计算属性的加载状态显示
- **错误处理** - 异步计算失败时的错误展示
- **进度指示** - 异步操作的进度显示

### 与 AutoStore 深度集成
- **计算属性自动绑定** - 利用 AutoStore 的 scope 机制
- **watch 监听器** - 支持 watch 回调
- **事件系统** - 集成 AutoStore 的事件总线

### 开发体验
- **TypeScript 完整支持** - 类型推断和类型安全
- **调试模式** - 开发环境下的详细日志
- **错误提示** - 友好的错误信息和警告

## Anti-Features（不实现的功能）

### v1 范围外的功能
- **x-model 双向绑定** - 留待 v2
- **组件系统** - 保持简单，指令级即可
- **插槽系统** - 不需要，保持轻量
- **自定义指令 API** - v1 不开放，仅内置指令
- **teleport 传送门** - 复杂度过高
- **transition 过渡动画** - 可用 CSS 实现
- **路由集成** - 不在模板引擎职责范围

### SSR 相关
- **服务端渲染** - v1 仅支持浏览器环境
- **hydration** - 无 SSR 逻辑，不需要水合

### 高级功能
- **指令优先级** - x-if 和 x-for 不支持同时使用（通过文档约束）
- **指令修饰符** - 简化语法，不支持 .prevent、.stop 等
- **过滤器** - 过期模式，用计算属性替代

## 功能复杂度评估

| 功能 | 复杂度 | 说明 |
|------|--------|------|
| x-text | 低 | 简单的文本内容更新 |
| x-html | 低 | innerHTML 设置 |
| x-visible | 低 | 样式切换 |
| x-if | 中 | 需要创建/移除 DOM 节点 |
| x-for | 高 | 需要跟踪列表项，key 管理 |
| 事件绑定 | 中 | 需要正确处理 this 和事件对象 |
| 属性绑定 | 中 | 需要区分不同属性类型 |
| 类/样式绑定 | 中 | 需要多种语法支持 |

## 功能依赖关系

```
核心引擎 (指令扫描、依赖追踪)
    ↓
基础指令 (x-text, x-html, x-visible)
    ↓
条件指令 (x-if) ← 依赖基础指令
    ↓
列表指令 (x-for) ← 依赖基础指令、key 机制
    ↓
事件绑定 ← 依赖核心引擎
    ↓
属性绑定 ← 依赖核心引擎
    ↓
类/样式绑定 ← 依赖属性绑定
```

## Table Stakes 标准

用户期望一个响应式模板引擎至少具备：
1. ✅ 条件渲染能力（x-if）
2. ✅ 列表渲染能力（x-for）
3. ✅ 内容绑定能力（x-text, x-html）
4. ✅ 事件处理能力（x-click 等）
5. ✅ 属性绑定能力（x-bind）
6. ✅ 样式控制能力（x-class, x-style）
7. ✅ 响应式自动更新

如果不具备以上功能，用户会认为这是一个不完整的产品。
