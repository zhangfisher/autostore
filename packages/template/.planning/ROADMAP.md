## 阶段 1：核心基础设施

**目标：** 建立 AutoRender 类架构、指令系统、模板扫描器

### 需求映射

- CORE-01: AutoRender 类初始化 API
- CORE-02: app.start() 方法
- CORE-03: app.stop() 方法
- CORE-04: app.destroy() 方法

### 成功标准

1. 用户可以通过 `new AutoRender(el, store, options)` 创建实例
2. 调用 `app.start()` 成功扫描模板并建立依赖
3. 调用 `app.stop()` 移除所有监听器，状态变化不再触发更新
4. 调用 `app.destroy()` 完全清理资源

### 主要任务

1. 创建 AutoRender 类骨架
2. 实现 DirectiveRegistry 指令注册系统
3. 实现 TemplateScanner 模板扫描器
4. 实现指令优先级排序
5. 建立与 AutoStore 的集成（watch 系统）

## 阶段 2：表达式系统

**目标：** 实现表达式解析、求值和依赖追踪

### 需求映射

- EXPR-01: 标识符访问
- EXPR-02: 字面量
- EXPR-03: 表达式安全性

### 成功标准

1. 表达式 `user.name` 正确访问嵌套属性
2. 表达式 `'hello'`、`123`、`true` 作为字面量返回
3. 不安全的表达式（如函数调用）抛出错误
4. 自动追踪表达式依赖的路径
5. 依赖变化时触发重新求值

### 主要任务

1. 创建 ExpressionParser 表达式解析器
2. 实现标识符访问语法
3. 实现字面量解析
4. 实现依赖提取逻辑
5. 实现表达式安全性验证
6. 集成到 TemplateScanner

### UI 提示：否

## 阶段 3：基础指令

**目标：** 实现内容绑定和显示控制指令

### 需求映射

- TEXT-01: x-text 指令 - 文本内容
- HTML-01: x-html 指令 - HTML 内容
- VISIBLE-01: x-visible 指令 - 显示/隐藏
- PERF-01: 细粒度更新
- DEV-02: 错误提示（部分）

### 成功标准

1. `<span x-text="user.name">` 正确显示文本并自动更新
2. `<div x-html="content.html">` 正确渲染 HTML（含安全警告）
3. `<div x-visible="isActive">` 正确控制 display 属性
4. 状态变更只更新相关元素，不重新扫描整个 DOM
5. 无效指令抛出清晰的错误信息

### 主要任务

1. 实现 XTextDirective
2. 实现 XHtmlDirective（含 XSS 检测）
3. 实现 XVisibleDirective
4. 实现指令清理机制（防止内存泄漏）
5. 实现路径索引（快速定位指令）
6. 添加基础错误提示

### UI 提示：是

本阶段实现基础的 UI 渲染能力，需要在真实 DOM 中验证。

## 阶段 4：控制流指令

**目标：** 实现条件和列表渲染

### 需求映射

- IF-01: x-if 指令 - 条件渲染
- IF-02: x-if 性能优化
- FOR-01: x-for 指令 - 列表渲染
- FOR-02: x-for key 属性
- FOR-03: x-for 响应式更新
- FOR-04: x-for 嵌套支持
- PERF-02: 初始渲染性能

### 成功标准

1. `<div x-if="condition">` 正确根据条件添加/移除元素
2. 条件变化时高效复用/清理 DOM
3. `<div x-for="item of items">` 正确渲染列表
4. 使用 `key` 属性时最小化 DOM 操作
5. 列表变化（新增/移除/排序）时正确更新
6. 支持嵌套列表（2 层），深层嵌套（>3 层）警告
7. 简单应用初始渲染 < 50ms

### 主要任务

1. 实现 XIfDirective
2. 实现 XForDirective
3. 实现 diff 算法（列表更新）
4. 实现 key 机制（唯一性验证）
5. 实现嵌套检测和警告
6. 实现条件切换的 DOM 复用
7. 性能优化和基准测试
8. 添加 x-if 和 x-for 冲突检测

### UI 提示：是

本阶段实现控制流指令，需要测试复杂的 DOM 结构。

## 阶段 5：交互指令

**目标：** 实现事件绑定、属性绑定、样式控制

### 需求映射

- EVENT-01: x-click 指令 - 点击事件
- EVENT-02: x-input 指令 - 输入事件
- EVENT-03: x-submit 指令 - 表单提交
- EVENT-04: 通用事件语法
- BIND-01: x-bind 指令 - 属性绑定
- BIND-02: 简写语法
- BIND-03: 布尔属性
- CLASS-01: x-class 指令 - 对象语法
- CLASS-02: x-class 指令 - 数组语法
- CLASS-03: x-style 指令 - 对象语法
- ASYNC-01: 异步计算状态反馈
- DEV-01: TypeScript 类型支持

### 成功标准

1. `<button x-click="increment()">` 点击时正确执行表达式
2. `<input x-input="value = $el.value">` 输入时正确更新
3. `<form x-submit="handleSubmit($event)">` 提交时正确调用
4. `<img :src="url">` 简写语法正常工作
5. `<button :disabled="!isValid">` 布尔属性正确设置
6. `<div x-class="{active: isActive}">` 对象语法正常工作
7. `<div x-style="{color: textColor}">` 样式正确应用
8. 异步计算属性的 isLoading/isError/isSuccess 状态正确反馈
9. 完整的 TypeScript 类型定义

### 主要任务

1. 实现 XOnDirective（支持 x-click、x-input、x-submit）
2. 实现通用事件语法（x-on:event）
3. 实现 $event、$el、$store 上下文变量
4. 实现 XBindDirective
5. 实现简写语法（:attr）
6. 实现布尔属性处理
7. 实现 XClassDirective（对象和数组语法）
8. 实现 XStyleDirective
9. 实现异步状态反馈机制
10. 创建完整的 TypeScript 类型定义

### UI 提示：是

本阶段实现交互功能，需要测试用户交互场景。
