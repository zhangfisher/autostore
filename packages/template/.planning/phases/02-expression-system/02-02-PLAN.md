  <behavior>
    - BindingManager.register(el, binding) 后，getByElement(el) 返回包含该 binding 的 Set
    - 同一元素注册多个 binding（多指令场景），getByElement 返回 Set 大小正确（D-11）
    - 元素被 GC 时对应绑定自动清理（WeakMap 特性，D-12——测试用 cleanupElement 显式验证）
    - cleanupElement(el) 调用该元素所有 binding 的 destroy() 并从 WeakMap 移除
    - destroyAll() 遍历所有元素调用 cleanupElement，最终 WeakMap 为空
    - getAll() 返回所有 binding 的扁平数组（供调试/日志）
    - 元素无绑定时 getByElement 返回 undefined
  </behavior>

  <action>
    创建 src/BindingManager.ts，导出 `class BindingManager`（对应 D-09~D-13）：

    1. 私有字段 `#bindings: WeakMap<HTMLElement, Set<Binding>>`（D-10 按元素分组 + D-12 WeakMap 防泄漏）。

    2. `register(el: HTMLElement, binding: Binding): void`：若 WeakMap 无该 el 则 `set(el, new Set())`，再 `get(el).add(binding)`。

    3. `getByElement(el: HTMLElement): Set<Binding> | undefined`：返回 WeakMap.get(el)。

    4. `cleanupElement(el: HTMLElement): void`（D-25~D-28）：取 Set，遍历调用每个 binding.destroy()（Binding.destroy 幂等，重复安全），再 `weakMap.delete(el)`。

    5. `destroyAll(): void`：WeakMap 无法直接遍历——因此额外维护一个 `#elementRefs: Set<HTMLElement>`（强引用）用于遍历，遍历每个 el 调用 cleanupElement，最后清空 #elementRefs。注意：#elementRefs 仅用于 destroyAll 遍历，不影响 WeakMap 的弱引用语义（元素被移除时 WeakMap 自动释放，#elementRefs 中可能残留但下次 register/destroyAll 时清理）。

    6. `getAll(): Binding[]`：遍历 #elementRefs 收集所有 Set 的 binding 返回扁平数组。

    创建 src/__tests__/BindingManager.test.ts：用真实 DOM（jsdom 或 bun 内置 DOM——bun 支持 document API）创建元素，mock Binding 对象（含 spy 的 destroy），验证 <behavior>。bun 运行时提供 document 全局对象，可直接 `document.createElement('div')`。

    切勿：不要在此修改 AutoTemplate.ts（Task 2 负责）。不要实现具体指令。
