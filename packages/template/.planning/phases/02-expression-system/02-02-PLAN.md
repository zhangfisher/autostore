    - 元素无绑定时 getByElement 返回 undefined
    - register 时若 binding.valid === false，可选择仍注册（destroy 时安全）或跳过——本任务选择注册（统一生命周期管理），由 Binding.bind() 内部静默跳过无效绑定
  </behavior>

    1. 私有字段 `#bindings: WeakMap<HTMLElement, Set<Binding>>`（D-13 按元素分组 + D-15 WeakMap 防泄漏）。

    2. `register(el: HTMLElement, binding: Binding): void`：若 WeakMap 无该 el 则 `set(el, new Set())`，再 `get(el).add(binding)`。

    3. `getByElement(el: HTMLElement): Set<Binding> | undefined`：返回 WeakMap.get(el)。

    4. `cleanupElement(el: HTMLElement): void`（D-25~D-28）：取 Set，遍历调用每个 binding.destroy()（Binding.destroy 幂等，重复安全），再 `weakMap.delete(el)`。从 #elementRefs 中也移除该 el。

    5. `destroyAll(): void`：WeakMap 无法直接遍历——因此额外维护一个 `#elementRefs: Set<HTMLElement>`（强引用）用于遍历，遍历每个 el 调用 cleanupElement，最后清空 #elementRefs。注意：#elementRefs 仅用于 destroyAll 遍历，不影响 WeakMap 的弱引用语义（元素被移除时 WeakMap 自动释放，#elementRefs 中可能残留但下次 register/destroyAll 时清理）。

    6. `getAll(): Binding[]`：遍历 #elementRefs 收集所有 Set 的 binding 返回扁平数组。
