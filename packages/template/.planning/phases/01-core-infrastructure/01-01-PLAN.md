    2. **scan(el: HTMLElement): DirectiveBinding[] 方法**
       - 清空已扫描记录：this.#scannedElements = new WeakMap()
       - 创建结果数组：const bindings: DirectiveBinding[] = []
       - 调用 walk(el, bindings) 遍历 DOM
       - 返回所有找到的指令绑定
    
    3. **walk(el: HTMLElement, bindings: DirectiveBinding[]): void 方法（私有）**
       - 使用 TreeWalker API：const walker = document.createTreeWalker(el, NodeFilter.SHOW_ELEMENT)（D-10）
       - 声明 node: Node | null = walker.currentNode
       - while (node !== null) 循环
       - 如果 node 是 HTMLElement（node instanceof HTMLElement）
         - 跳过已扫描元素：if (this.#scannedElements.has(node)) { continue }
         - 标记为已扫描：this.#scannedElements.set(node, true)
         - 调用 extractDirectives(node, bindings) 提取指令
       - node = walker.nextNode() 继续遍历
    
    4. **extractDirectives(el: HTMLElement, bindings: DirectiveBinding[]): void 方法（私有）**
       - 获取所有属性：Array.from(el.attributes)
       - 遍历属性，找到以 x- 开头的指令属性
       - const directivePattern = /^x-(\w+)(?::|$)/ 匹配指令名
       - 对于每个匹配的属性
         - 提取指令名：const match = attr.name.match(directivePattern)
         - const directiveName = match[1].toLowerCase()
         - 检查指令是否在注册表中：if (this.#registry.has(directiveName))
           - 创建绑定：bindings.push({ directive: directiveName, expression: attr.value, element: el, parsed: null })
         - 如果指令不存在，跳过（不抛出错误，符合 D-11 静默失败策略）
    
    5. **扫描新元素方法 scanNew(elements: HTMLElement[]): DirectiveBinding[]**
       - 接受新增元素数组（增量扫描支持，D-08）
       - const newBindings: DirectiveBinding[] = []
       - 遍历 elements，对每个元素调用 walk(element, newBindings)
       - 返回新绑定
    
    D-08 实现：增量扫描策略，scanNew 方法扫描新增元素
    D-09 实现：WeakMap 在实例级别追踪已扫描元素
    D-10 实现：使用 TreeWalker API 高效遍历
    D-11 实现：未知指令静默跳过（不抛出错误）
  </action>
